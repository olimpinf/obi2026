A seletiva para as olimpíadas internacionais usa como referência a Ementa da Olimpíada Internacional de Informática.

Competidores devem utilizar a linguagem C++ e estar familiarizados com todos os conteúdos da ementa da Modalidade Programação, assim como os conteúdos adicionais listados abaixo.

1. Fundamentos de Matemática

Conceitos de Matemática Discreta

  • Probabilidade discreta

Conceitos de Grafos

  • Grafos planares: fórmula de Euler

2. Fundamentos de Computação

Informática Básica

  • Interfaces de linha de comando

Programação

  • Bitsets
  • Geradores de números aleatórios da biblioteca padrão
  • Redirecionamento de entrada/saída para arquivos
  • Programação orientada a eventos: algumas tarefas (“interativas” / “comunicativas”) podem envolver interação com um ambiente reativo. O competidor deve ser capaz de:
    • Utilizar bibliotecas específicas da competição de acordo com instruções fornecidas durante a prova
    • Implementar a interação com o ambiente fornecido

3. Algoritmos e Estruturas de Dados

Estratégias de Algoritmos

  • Heurísticas
  • Algoritmos randomizados

Estruturas de Dados

  • Decomposição avançada:
    • Decomposição de estruturas dinâmicas com rebalanceamento
    • Algoritmo de Mo
  • Árvore de Segmentos 1D Dinâmica
  • Estruturas persistentes (por exemplo, Árvore de Segmentos Persistente)
  • Árvore de Busca Binária Balanceada (Treap, Splay Tree ou equivalente) 1D
  • Técnica de Divisão-e-Conquista para responder consultas em vetor estático
  • Union-Find Semi-persistente, Union-Find com Rollback
  • Trie de Bits

Algoritmos de Ordenação e Busca

  • Quicksort e Quickselect

Algoritmos de Matemática

  • Operações simples (adição, subtração e multiplicação) em inteiros de tamanho arbitrário (Big Num)

Algoritmos de Programação Dinâmica

  • Programação dinâmica em inteiros de tamanho arbitrário (Digit DP)
  • Otimizações avançadas de programação dinâmica:
    • Truque do Fecho Convexo (CHT)
    • Otimização por Divisão-e-Conquista
    • Otimização de Knuth

Algoritmos em Grafos

  • 2-SAT (com componentes fortemente conexas)
  • Máximo conjunto de arestas independentes em um grafo bipartido (Bipartite Matching) em O(VE)
  • Biconectividade em grafos não-direcionados (Bridge Tree, Block-Cut Tree)

Algoritmos em Árvores

  • Decomposição em Centroides
  • Decomposição Heavy-Light

Algoritmos de Geometria

  • Checar se um ponto está em um polígono convexo em O(log N)

Tópicos Explicitamente Excluídos

(Seção interna apenas para referência, não publicar)

Fundamentos de Matemática

Conceitos de Aritmética e Geometria

  • Geometria em três ou mais dimensões
  • Análise e aumento da precisão de cálculos com pontos flutuantes
  • Divisão modular e elementos inversos
  • Números complexos
  • Cônicas gerais (parábolas, hipérboles, elipses)
  • Funções trigonométricas

Fundamentos de Lógica

  • Formas normais
  • Validade
  • Limitações de lógica de predicados

Conceitos de Matemática Discreta

  • Cardinalidade e contabilidade de conjuntos infinitos
  • Solução de relações de recorrência linear
  • Lema de Burnside

Conceitos de Grafos

  • Hipergrafos
  • Classes específicas de grafos como grafos perfeitos
  • Parâmetros estruturais de grafos como treewidth e expansion
  • Testes de planaridade
  • Encontrar separadores para grafos planares

Outros Tópicos Excluídos

  • Álgebra linear, incluindo:
  • Multiplicação, exponenciação e inversão de matrizes e eliminação Gaussiana
  • Fast Fourier Transform
  • Cálculo
  • Teoria de jogos combinatoriais, como NIM e Sprague-Grundy
  • Estatística

Fundamentos de Computação

Programação

  • Cálculos não-triviais com floating point numbers; lidando com erros de precisão
  • Programação orientada a objetos
  • Arquitetura de software

Algoritmos e Estruturas de Dados

Fundamentos de Análise de Algoritmos

  • Análise assintótica de complexidade média
  • Usar relações de recorrência para analisar a complexidade de algoritmos recursivos
  • Conceitos de computabilidade
  • Classe de complexidade NP, P vs NP e provas de NP-Hardness (um problema NP-hard ainda pode aparecer como problema de otimização, mas não deve exigir conhecimentos teóricos em Teoria da Complexidade)

Estratégias de Algoritmos

  • Algoritmos de Agrupamento (clustering, por exemplo, k-means, k-nearest neighbor)
  • Minimizar funções multivariadas usando métodos numéricos
  • Problemas de otimização cuja análise mais natural envolve matroides
  • Problemas baseados em interseção de matróides (exceto Bipartite Matching)
  • Alpha-beta pruning

Estruturas de Dados

  • Implementação de hash tables e estratégias para resolução de colisões
  • Estruturas e algoritmos de strings (KMP, Rabin-Karp hashing, suffix arrays/trees, suffix automata, Aho-Corasick) exceto Trie
  • Variantes complexas de Heap como Binomial Heap e Fibonacci Heap
  • Estruturas de árvore multidimensionais quando usadas para responder queries multidimensionais (por exemplo, Segment Tree 2D ou uma Treap de Treaps)
  • Estruturas persistentes com fat nodes ou outros métodos mais complicados

Algoritmos de Matemática

  • Eliminação Gaussiana
  • Fast Fourier Transform (FFT)

Algoritmos em Grafos

  • Algoritmos de Min-Cut / Max-Flow
  • Lexicographical BFS e Maximum Adjacency Search

Algoritmos em Árvores

  • Estruturas para árvores dinâmicas (por exemplo, Link-Cut Tree)

Algoritmos de Geometria

  • Dualidade ponto-reta
  • Interseção de semiplanos, diagramas de Voronoi e triangulação de Delaunay
  • Interseção de círculos com retas e círculos
  • Programação linear em 3 ou mais dimensões e suas propriedades geométricas
  • Centro de massa de um objeto 2D
  • Composições de transformações geométricas cuja análise é mais simples com álgebra linear
